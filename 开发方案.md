# 🎋 DragonBoatFestival（端午节活动插件）开发方案

## 一、项目定位

插件名称：DragonBoatFestival

适用版本：

* Spigot 1.12.2
* Paper 1.12.2

开发语言：

* Java 8

兼容插件：

* Vault（可选）
* PlaceholderAPI（可选）
* TrMenu（可选）
* ajLeaderboards（通过 PlaceholderAPI 占位符适配）

设计目标：

* 独立运行
* 无需依赖菜单插件
* 配置简单
* 低性能占用
* 适合生存服、轻RPG服

---

# 二、核心玩法

## 1. 粽子材料收集

玩家通过日常行为获得材料。

材料类型：

### 糯米

获取方式：

* 挖矿概率掉落（任何 ore）
* 打怪概率掉落

### 粽叶

获取方式：

* 破坏树叶概率掉落（任何 leaves）

### 红枣

获取方式：

* 采集作物概率掉落（小麦/胡萝卜/马铃薯/甜菜根/西瓜/南瓜）

### 鲜肉

获取方式：

* 击杀动物概率掉落

---

## 数据结构

```java
// 材料以背包物品形式存在（lore 标记），不占 PlayerData 字段
// lore 会说明材料可合成粽子、间接食用、兑换奖励等用途
MaterialItems
{
    create(type, amount)    // 生成带标记和说明 lore 的材料物品
    give(player, type, amt) // 给予玩家材料
    count(player, type)     // 统计背包中的指定材料数量
    has(player, cost)       // 检查背包材料是否足够
    take(player, cost)      // 从背包扣除材料
}

// 玩家持久化数据（YAML 存储）
PlayerData
{
    points;         // 节日积分
    normalMade;     // 普通粽子制作数
    luxuryMade;     // 豪华粽子制作数
    fishRewards;    // 钓鱼奖励次数
    shopPurchases;  // 商店兑换次数
    signDays;       // 累计签到天数
    lastSign;       // 最后签到日期
}
```

# 2. 粽子制作

命令：

/duanwu make [normal|luxury]

或

GUI制作（/duanwu 主菜单 → 制作粽子）

制作类型：

## 普通粽子

消耗：

* 糯米 ×5
* 粽叶 ×2

获得：

* 节日积分 ×1
* 普通粽子物品（可右键食用/交易）

---

## 豪华粽子

消耗：

* 糯米 ×10
* 粽叶 ×5
* 红枣 ×3
* 鲜肉 ×3

获得：

* 节日积分 ×5
* 豪华粽子物品（可右键食用/交易）

---

# 3. 节日积分系统


```text

```

# 4. 活动商店

命令：

```text
/duanwu shop
```

打开 GUI 商店，商品从 `shop.yml` 配置（支持自定义物品名、Lore、积分价格、执行命令）。

---

# 5. 幸运摸鱼系统

监听：

```java
PlayerFishEvent
```

概率奖励：

| 奖励   | 概率  |
| ---- | --- |
| 糯米   | 40% |
| 粽叶   | 30% |
| 红枣   | 15% |
| 鲜肉   | 10% |
| 幸运礼包 | 4%  |
| 传说礼包 | 1%  |

---

# 6. 每日签到

命令：

```text
/duanwu sign
```

每天一次，5 天循环。

记录：

```java
lastSign  // 日期字符串
signDays  // 累计天数 (0-4 循环)
```

奖励在 config.yml 中配置，每轮包含 points/exp/commands。

---

# 7. BOSS 活动

管理员可开启端午 BOSS：

```text
/duanwu boss spawn
```

可配置：实体类型、血量、名称、奖励积分、击杀命令。

BOSS 用 Metadata 标记，死亡时发放奖励并推进全服目标。

```yaml
  enabled: true
```

# 五、数据存储

方式：

YAML

目录：

```text
plugins/
└─ DragonBoatFestival/
   ├─ config.yml
   ├─ messages.yml
   ├─ shop.yml
   ├─ goals.yml
   └─ playerdata/
       └─ UUID.yml
```

玩家数据示例：

```yaml
name: PlayerName
points: 50
stats:
  normalMade: 10
  luxuryMade: 3
  fishRewards: 5
  shopPurchases: 2
  signDays: 3
lastSign: 2026-06-08
```

材料以背包物品形式保存（通过 lore 标记识别），不在 YAML 中存储。

# 八、全服任务系统

在 config.yml 中配置全服目标：

```yaml
server-goals:
  make-zongzi:
    enabled: true
    name: '全服制作粽子'
    target: 500
    complete-message: '...'
    commands:
      - 'broadcast ...'
  sign:
    enabled: true
    name: '全服每日签到'
    target: 100
    ...
  boss-kill:
    enabled: true
    name: '击败端午 BOSS'
    target: 1
    ...
```

所有玩家共同推进任务，完成后全服广播并执行命令。

数据持久化到 goals.yml，支持脏标志延迟批量写入。

# 九、PlaceholderAPI 占位符

如果服务器安装 PlaceholderAPI，注册变量：

```text

%duanwu_signed_today%     # 今日是否签到 (yes/no)
%duanwu_fish_rewards%     # 钓鱼奖励次数
%duanwu_shop_purchases%   # 商店兑换次数
%duanwu_rice%             # 糯米背包数量
%duanwu_leaf%             # 粽叶背包数量
%duanwu_jujube%           # 红枣背包数量
%duanwu_meat%             # 鲜肉背包数量
%duanwu_materials_total%  # 材料总数（四项之和）
```

# 十一、性能设计

* 所有数据缓存内存（ConcurrentHashMap 线程安全）
* 定时异步保存，线程安全的批量写入
* 玩家退出自动保存（主线程操作，避免数据竞争）
* 全服任务使用脏标志延迟写入磁盘
* 不使用 NMS
* 不依赖数据库

预计性能消耗：

在线100人以下：

TPS影响接近0

适合长期活动服使用。

# 十三、精品版升级方案

目标：把插件从"基础活动"升级为"玩家不用问、服主好配置、活动有目标感"的精品节日插件。

## 1. 玩家体验升级（✅ 已完成）

### 活动中心主菜单

玩家输入：

```text
/duanwu
```

打开 GUI：

```text
🎋 端午活动中心
├─ 我的状态：积分、材料、签到状态
├─ 制作粽子：普通/豪华粽子 GUI 制作
├─ 活动商店：积分兑换奖励
├─ 每日签到：点击领取每日奖励
├─ 幸运摸鱼：查看钓鱼奖励说明
└─ 新手指南：一步步告诉玩家怎么玩
```

### 新手引导

首次进入服务器时，如果活动开启，延迟发送小白引导：

```text
欢迎参加端午活动！
第一步：挖矿、破坏树叶、收作物、打怪、钓鱼获得材料。
第二步：输入 /duanwu 打开活动中心。
第三步：点击"制作粽子"获得积分。
第四步：点击"活动商店"兑换奖励。
```

同时提供：

```text
/duanwu guide
```

展示完整游玩闭环和常见问题。

### 玩家状态页

提供：

```text
/duanwu status
```

显示：

* 节日积分
* 背包内端午材料数量
* 今日是否已签到
* 下一步推荐操作
